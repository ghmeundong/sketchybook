웹 게임 개발

-Sketchybook-

[게임 컨셉]
- 기본 구조: 공을 굴려 별을 먹는 퍼즐
- 플레이 방식: 사용자가 화면에 선(Line)을 그려서 공의 진행 경로를 유도
- 차별화 요소:
  - 자기장: 특정 구간에서 공이 자석에 끌리거나 밀려남
  - 바람: 일정 방향으로 힘을 가해 공을 띄우거나 밀어냄
  - 물: 공이 물 위에 뜨거나 물살에 휩쓸림
  - 쇠사슬/스프링: 오브젝트를 연결해 유연한 움직임 구현

[플랫폼/배포]
- 프론트엔드: GitHub Pages로 정적 배포
- 백엔드:
  - 캐주얼 싱글플레이 → 불필요
- 서버리스 스토리지:
  - Cloudflare KV/Durable Objects → 진행 저장

[로그인/저장]
- 단순 점수 저장:
  - LocalStorage 기반
  - 같은 브라우저/기기에서만 유지
- 확장 기능:
  - 구글 OAuth 로그인 → UI를 게임에 맞게 커스텀
  - 리디렉션 방식으로 인증 후 토큰 전달
  - Supabase 연동 시 유저별 점수/스테이지 공유 가능

[UI/연출]
- roughjs로 손그림풍 UI 구현
- 종이 질감 배경 이미지 활용
- 로그인 버튼:
  - 구글 로그인 버튼을 게임 UI에 맞게 커스텀

[물리 엔진]
- Matter.js:
  - 가볍고 브라우저 친화적 → 캐주얼 퍼즐에 최적
  - force 적용으로 바람/자석 효과 구현 가능
  - constraints로 쇠사슬, 스프링 구현 가능

  [구현 목표]
- 선(Line) 처리:
  - 오브젝트와 겹칠 경우 → 합쳐져 고정
- 화면 비율:
  - 기본 기준은 16:9
  - 세로(height)를 기준으로 논리 좌표계 설정
  - 가로(width)는 여백 처리, 울트라와이드 모니터는 배경으로 채움
- 논리 좌표계:
  - 내부 좌표계를 고정 1600:900
  - 실제 화면 크기에 맞춰 scaleX, scaleY로 변환
  - 오브젝트 위치/크기는 논리 좌표계 기준으로 계산
- 모바일:
  - 강제로 가로 모드(90도 회전) 고정
  - 화면 비율 편차는 스케일링으로 대응

[향후 확장 아이디어]
- 스테이지 에디터: 유저가 직접 퍼즐을 만들고 공유
- 글로벌 랭킹: 점수 경쟁 기능 추가
- 협력 플레이: Supabase 연동으로 스테이지 풀기/공유
